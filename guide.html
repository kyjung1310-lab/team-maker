3개 페이지 모두 동일한 방식입니다. 비밀번호 없이 본인 정보만 입력합니다.

① GitHub 팀 저장소

팀이 코드·문서를 함께 저장하는 공간. 매주 실습 결과물을 올립니다(commit·push).

1주차 준비 (팀당 1회)

  1. 팀 대표가 GitHub에서 팀 저장소 생성 (교수님이 미리 만들 수도 있음)
  2. 팀원을 협업자로 추가: 저장소 → Settings → Collaborators

저장소 주소 등록 (repo.html) — 아주 쉬움

  1. 로그인 후 "우리 팀 GitHub 저장소" 카드 확인
  2. 📌 교수 사전 등록 / ✓ 등록됨 / 미등록 상태가 표시됨
  3. 미등록이면 다음 중 편한 방법으로 입력 후 저장:

✨ 가장 쉬운 방법 — 계정명만 입력

팀 대표의 GitHub 계정명만 입력하면 (예: hong-gildong) 권장 저장소명이 자동으로 붙어 https://github.com/hong-gildong/과목-team5 처럼 완성됩니다. 입력하는 동안 최종 주소 미리보기가 표시됩니다.

이미 만든 저장소가 있으면 전체 주소를 그대로 붙여넣기 하면 됩니다 (/tree/main 같은 뒤 경로는 자동 정리).

💡 같은 팀원이면 누구나 등록·수정 가능, 변경 시 모든 팀원에게 자동 반영됩니다.

⚠️ 저장소 용량 규약 — 꼭 지키기

GitHub에는 코드·문서(텍스트)만 올립니다. 영상·대용량 파일을 직접 넣으면 용량 한도(팀당 약 1GB 권장)를 금방 초과합니다.

✅ 저장소에 올림

소스 코드 (.py·.js·.java …) · 문서 (.md·.txt·소형 docx/pdf) · 설정 파일

❌ 안 올림 → 외부 링크

영상·대형 PPT·이미지·데이터셋·빌드 산출물 → Drive/Release에 올리고 링크만

💡 텍스트는 15주 누적해도 용량이 거의 안 늘어납니다. 대용량 바이너리만 빼면 저장소가 가볍게 유지됩니다.

💬 commit 메시지 형식

[W3] 컨테이너 실습 완료   ·   형식: [W{주차}] 요약

⏰ 매주 18:20 마감

실습 종료 시각까지 commit·push 완료. 결석 시 git clone 후 본인 PC에서 재현하여 보고서 제출.

Git 기본 명령 (터미널)

# 처음 한 번 — 저장소 복제
git clone https://github.com/계정/저장소.git

# 매주 작업 후
git add .
git commit -m "[W3] 실습 내용 요약"
git push

💡 명령어가 어렵다면 GitHub Desktop 앱으로 클릭만으로 가능합니다.

② 과제 제출 (Smart Submission)

submit.html에서 주차별 보고서·과제를 제출합니다. 공식 평가 채널입니다.

주차 버튼 색상 = 마감 상태

⚪ 회색정상 — 여유 있음
🟡 노랑주의 — 마감 3일 이내
🟠 주황 ⚠임박 — 마감 24시간 이내
🔴 빨강마감 후 — 지연 제출 가능(늦음 표시)
⛔ 취소선마감 — 제출 불가

제출 순서

  1. 로그인 → 주차 버튼 클릭
  2. 파일 박스 클릭 또는 끌어다 놓기 (최대 100MB, .exe·.bat 제외)
  3. 메모(선택) 입력 후 제출하기
  4. 하단 "제출 내역"에서 확인 · 본인/팀원 토글 · 받기로 재다운로드

🎬 시연 영상

100MB 이하 → 직접 첨부 / 초과 시 → GitHub Release·Google Drive에 올린 후 링크를 문서에 담아 제출

🔄 같은 주차에 다시 제출하면 이전 파일이 자동 교체되어 최신 1개만 남습니다 (안심하고 수정 제출 가능).

③ 설문 / 피드백

survey.html에서 수업 평가·동료 평가·설문에 응답합니다.

  1. 로그인 → "참여 가능한 설문"에서 설문 클릭
  2. 척도(1~5점 버튼) · 객관식 · 서술형 · 팀원 평가에 응답
  3. 응답 제출 클릭

🅰️🅱️ 팀원 평가

팀원 이름이 자동 표시됩니다(직접 입력 안 함). 짝 A/B로 나뉘며 본인은 제외. 진행도(2/3명)로 확인.

🔒 익명 설문

"익명" 표시가 있으면 누가 응답했는지 저장되지 않습니다. 솔직하게 응답하세요.

파일명·작성 규약 (꼭 지키기)

종류형식 / 예시
개인 보고서학번_이름_W{주차}_실습보고서.docx
예: 2022001234_홍길동_W3_실습보고서.docx
팀 산출물팀명_W{주차}_{유형}.docx
예: 5팀_W3_과제1.docx
GitHub commit[W{주차}] 요약

자주 묻는 질문

Q. 비밀번호가 따로 있나요?

없습니다. 이름(ID) + 학번(PW)으로 본인 확인합니다.

Q. "명단에서 찾을 수 없습니다"가 떠요.

① 이름·학번 오타 ② 팀 잘못 선택 ③ 명단 미등록. 확인 후에도 안 되면 교수님께 문의.

Q. 파일이 100MB를 넘어요.

영상은 GitHub Release·Google Drive에 올리고 링크로 제출하세요.

Q. 마감이 지났는데 제출할 수 있나요?

🔴(지연 가능)이면 제출되나 늦음 기록, ⛔(마감)이면 불가. 교수님께 문의.

Q. 팀이 바뀌었어요.

교수님이 명단 수정 시 자동 반영됩니다. 이전 제출물도 새 팀으로 함께 이동.

Q. 한 번 제출한 파일을 다시 올릴 수 있나요?

같은 주차에 다시 제출하면 됩니다. 제출 내역에서 확인하세요.

Q. 모바일로도 되나요?

됩니다. 스마트폰 브라우저에서도 동일하게 작동합니다.

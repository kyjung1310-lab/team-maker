매주 commit·push 가이드

⏰ 매주 18:20 마감

실습 종료 시점까지 팀 저장소에 commit·push 의무. 결석자는 clone으로 본인 환경에서 재현 후 보고서 제출.

⚠️ 저장소 용량 규약 — 꼭 지키기

GitHub 저장소에는 코드·문서(텍스트)만 올립니다. 영상·대용량 파일을 저장소에 직접 넣으면 용량 한도(팀당 1GB 권장)를 금방 초과합니다.

✅ 저장소에 올림

소스 코드(.py·.js·.java 등)
문서(.md·.txt·소형 .docx/.pdf)
설정 파일

❌ 저장소에 안 올림 → 외부 링크

영상(.mp4 등)·대형 PPT·이미지
데이터셋·빌드 산출물(node_modules 등)
→ Drive/GitHub Release에 올리고 링크만

💡 텍스트(코드·문서)는 15주 누적해도 용량이 거의 안 늘어납니다. 대용량 바이너리만 빼면 저장소가 가볍게 유지됩니다.

📁 권장 폴더 구조

team-repo/
├ week01/ ← 코드·문서
├ week02/
├ ...
├ week15/
├ links.md ← 영상·대용량 외부 링크 모음
└ .gitignore ← 빌드물·대용량 자동 제외

🚫 .gitignore 예시 (대용량 자동 제외)

*.mp4
*.mov
*.zip
node_modules/
dist/ build/

💬 commit 메시지 형식

[W{주차}] {간단한 요약}

예: [W3] 컨테이너 실습 완료
